# AI Usage Log: MAZ-192 Show level names clearly in preview cards (client)

## Task / Problem

The level list cards only showed the order number, difficulty stars and a timed badge,
so a player could not tell which level a card represents by name. MAZ-192 makes every
level preview/card identify the level with its product name, for both the backend
catalog and the offline fixture fallback, readable on small screens.

## Result Obtained

- `LevelCard` now renders `level.name` (one line, `numberOfLines={1}`, centered, small
  font) between the order number and the difficulty stars; the timed badge folds inline
  with the stars to keep the fixed-height tile uncluttered. The accessibility label now
  leads with the name.
- Order/difficulty metadata preserved. No data change — `LevelListItem.name` was
  already populated from both the backend catalog and the fixtures.
- No new layers/patterns; presentation only. AGENTS architecture rules unchanged.

## Lessons / Limitations

- The name source is the catalog name already on `LevelListItem` (backend
  `LevelCatalogSummary.name` online, `manualLevels.ts` draft names offline), so a card
  always has a readable name — no new fallback needed.
- Truncation is verified via the rendered `numberOfLines` prop; true on-device wrap/
  overflow still benefits from a manual `expo start` check on a narrow screen.
